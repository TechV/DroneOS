IA 498 – Independent Study, Summer 2014

Student: Ryan Scott

Professor: Samir Tout

Eastern Michigan University


Overview:

The goal of this project is to create a well documented method to create a stable flying platform that is controlled via a Raspberry Pi. The design aspect will be adaptable to multiple uses ranging from video surveillance, to more advanced based applications, such as network or radio frequency mapping. The basic platform will consist of four motors and four electronic speed controllers. These will be controlled by a Raspberry Pi which will receive sensor information from an accelerometer and gyroscopic sensor. These sensors allow the Raspberry Pi to adjust the speeds on the motors to maintain stability with the horizon as well as allow for ratio based adjustments for bearing and movement. The software level of the project will consist of a wireless connection for streaming of video and manual controls to and from the device. Control methods will be relatively easy to map allowing for future projects to experiment with various other forms of controlling and interfacing with the device.

The platform will be pilot-able by use of a video headset and custom coded implementation enabling the use of a wired PS3 controller on the ground. The software system will be comprised of a custom Linux image for the Raspberry Pi as well as a live bootable controller system for a laptop.

Another future application could include experiments in “war-flying” simulations, to investigate the potential misuse of small scale mobile hacking stations.

The project will in its culmination become a detailed how-to guide in open-source flying platforms. Documentation will include pictures and videos as well as helpful lessons learned along the way to help future enthusiasts to achieve the basics of flight more easily, leaving more room for more advanced projects from the growing community.

The project will consist of a series of target goals and test cases as follows:

  • Capability 1: QC is assembled and parts are tested. Test Case 1: Basic control of the parts can be demonstrated.

  • Capability 2: QC can hover. Test Case 2: The QC provides enough thrust on each motor to get off the ground.

  • Capability 3: QC can be easily calibrated. Test Case 3: Hovering can now be sustained more stably and telemetry data is more accurate.

  • Capability 4: QC is capable of performing maneuvers. Test Case 4: QC can now rotate and move in a direction.

  • Capability 5: QC is capable of fine tuned maneuvers from external controls. Test Case 5: Live data from the controller can now control QC movement.

  • Capability 6: QC has streaming video. Test Case 6: QC can now be piloted by use of controller and video feed.

  • Capability 7: QC has some form of automation to procedures. Test Case 7: QC can replay a command set.

  • Capability 8: QC can be built by others by following video instructions. Test Case 8: Test cases 1-7 can be completed with minimal issues on replicated project.
